# pSEO Source Guardrails

These guardrails are mandatory for Niblah pSEO pages.

They are based on:

- `/Users/al/Downloads/Content guidelines (1).docx`
- `/Users/al/Downloads/Optimizing your website for generative AI features on Google Search (1).docx`

## 1. Non-Commodity Content

Do not publish pages that only restate common SEO advice.

Each page must include at least one non-commodity variable:

- local insight from the city or region
- Niblah-specific process or opinion
- mini-audit of the local SERP or competition
- real or defensible example
- comparison between options
- practical workflow
- customer-style scenario
- original data point or observation

Weak:

- "Wij helpen bedrijven in [plaats] met SEO, Google Ads en websites."

Stronger:

- "Voor bedrijven in Barneveld zien we dat lokale zichtbaarheid vaak splitst tussen Barneveld zelf, Harselaar en omliggende dorpen zoals Voorthuizen. Daarom bouwen we de pagina niet alleen rond 'marketing bureau Barneveld', maar ook rond servicegebieden, lokale bewijsvoering en duidelijke conversiepaden."

## 2. One Intent Per Page

Every page needs one primary intent.

Do not create separate pages for small keyword variations when the visitor wants the same thing.

Allowed:

- `/marketing/barneveld` for local marketing bureau intent.
- `/services/seo` for general SEO service intent.

Not allowed:

- `/marketingbureau-barneveld`
- `/online-marketing-bureau-barneveld`
- `/marketing-bedrijf-barneveld`

Those are the same intent and should be served by one strong page.

## 3. No Doorway Pages

A page fails if it is a thin doorway page whose only unique element is the location name.

Every local page needs:

- local context
- local business examples
- local service area details
- local internal links
- a unique intro and unique FAQ
- a unique proof section

## 4. Write For People First

The page must satisfy a real business owner.

The page should answer:

- Why should a company in this place trust Niblah?
- What problem does this page solve?
- What services are relevant here?
- What does the process look like?
- What should the visitor do next?

If the page feels made only for Google, do not publish it.

## 5. AI Search Is Still Search

Do not create separate "GEO" or "AEO" tricks for these pages.

Google's guidance says generative AI features use core Search systems. The answer is still strong SEO:

- valuable content
- clear structure
- crawlable pages
- good page experience
- helpful media
- reduced duplicate content
- real expertise and experience

## 6. Ignore Fake AI Hacks

Do not spend production time on:

- special AI-only markup for Google
- `llms.txt` as a Google ranking tactic
- artificial "chunking" for AI
- rewriting copy purely for AI systems
- fake mentions across the web
- schema spam

Structured data can be used where it is genuinely appropriate, but it is not a substitute for content quality.

## 7. Technical SEO Requirements

Every page must be:

- crawlable
- indexable
- linked internally
- included in sitemap
- self-canonical unless there is a clear exception
- mobile friendly
- fast enough to use comfortably
- free of obvious layout issues

The page should use semantic structure where practical:

- one H1
- logical H2/H3 hierarchy
- descriptive links
- meaningful image alt text where images are used

## 8. Internal Linking Requirements

Every page must link to:

- one relevant service page
- one relevant tool or resource when available
- two nearby location pages where relevant
- the contact page

Every new or upgraded page should also receive at least one inbound internal link from:

- the relevant service page
- a nearby location page
- a hub page or footer/location index where appropriate

## 9. Page Quality Gate

Before publishing, answer yes to all:

- Does this page have a unique proof variable?
- Would this page still be useful if Google did not exist?
- Is the primary intent clear within 5 seconds?
- Is there a concrete next step?
- Is the copy different in structure and substance from other local pages?
- Does it avoid fake claims and unverifiable numbers?
- Does it link users deeper into the site?

If any answer is no, keep it in draft.
